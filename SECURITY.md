# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities through one of these channels:

- GitHub Security Advisories for this repository
- Direct contact with repository maintainers

Include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Suggested remediation (if available)

## Scope

Security reports are especially valuable for:

- Authentication and authorization logic
- API endpoint abuse and injection risks
- Dependency vulnerabilities
- Secrets exposure
- Data privacy leaks involving health data

## Response Expectations

- Initial triage target: within 72 hours
- Status updates during investigation and fix
- Coordinated disclosure after a patch is available
